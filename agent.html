어떻게 제품이 되어 돌아오는가

이 팀의 개발은 사람이 코드를 치는 대신, 텔레그램 그룹에 붙은 AI 에이전트(봇)가 대화를 받아 직접 코드를 쓰고 배포합니다. 누구나 한 줄 코멘트로 제품을 바꿀 수 있어요. 아래는 그 전체 구조입니다.

💬 멤버가 그룹에 의견 → 🤖 에이전트가 맥락을 읽고 → 🔀 AI 일/사람 일을 가르고 → ⚙️ 코드·데이터를 만들고 → ✅ 진짜 화면으로 검증 → 🚀 배포 → 📣 보고. 이게 한 바퀴, 계속 반복됩니다.

에이전트가 보는 것 / 하는 것

👁 보는 것 (읽기)

매번 일하기 전에 이 자료들로 맥락을 잡는다.
🧭
지도 — project/ 의 trunk(큰 줄기)·MAP(현황)·branches(역할). 팀의 방향·결정이 여기 박혀 있음.
🧠
메모리 — 팀원·결정·일정 등 세션을 넘어 기억하는 사실.
📄
소스코드 — mvp/site (히트맵·대시보드·데이터)와 mvp/skills (AI 스킬 스펙).
🕓
git 이력 — 지금까지 무엇을 왜 바꿨는지 전체 기록.

🛠 하는 것 (실행)

사람의 승인 방향 안에서 직접 수행한다.
✍️
코드·데이터 작성 — 화면·로직·점수 데이터를 직접 만들고 고친다.
🔎
리서치 — 웹·공공데이터(법제처·DART 등)에서 사실을 찾아 확인.
🧩
스킬 실행 — 정책 추출·점수화·상세분석·출처 연결 등 재사용 능력 단위.
🚀
커밋·배포·검증 — git 기록 → 라이브 배포 → 브라우저로 실제 확인.
역할 분담 — 왜 이렇게 일하나
🙋 사람 (멤버)
  • 방향을 정한다 — 무엇을 만들지, 어디로 갈지
  • 도메인 정답을 쥔다 — 어느 기업이 수혜인지, 점수가 맞는지 (제품의 핵심 자산)
  • 평가한다 — AI 초안을 출처와 대조해 맞다/틀리다
  • 척도·가중치 같은 판단을 내린다
🤖 에이전트 (봇)
  • 실행한다 — 코드·화면·데이터를 만든다
  • 생성·검색·파싱 — 요약·서술·출처 찾기·문서 추출
  • 배포·검증하고 결과를 보고한다
  • 같은 일을 지치지 않고 반복한다
🔑 원칙: 결정·정답은 사람, 생성·실행·검증은 AI. 그래서 코딩을 몰라도 도메인·기획 강점만으로 제품을 빠르게 굴릴 수 있습니다. AI가 만든 건 항상 초안이고, 도메인 멤버의 평가가 제품의 진짜 가치를 만듭니다.
